Getting Started

Up and running in 5 steps

No custom server or complex deployment. Everything happens inside Power BI Desktop.

1

Install the Visual

Download the latest .pbiviz from Releases. In Power BI → Insert → More visuals → Import from file.

2

Load Your Data

Import the included PBI_Lineage_SampleData.xlsx to try instantly, or use the Power Query M scripts to call the PBI REST API / Admin Scanner API and build the Nodes and Edges tables automatically.

3

Create the NodesJSON Measure

Add this DAX measure to your model. Two options — both auto-detected:

DAX — CONCATENATEX 
NodesJSON =
VAR _rows =
  CONCATENATEX( Nodes,
    "{"
      & """NodeId"":"""   & SUBSTITUTE(Nodes[NodeId],"""","'") & ""","
      & """NodeName"":""" & SUBSTITUTE(Nodes[NodeName],"""","'") & ""","
      & """NodeType"":""" & Nodes[NodeType] & ""","
      & """Workspace"":"""& SUBSTITUTE(Nodes[Workspace],"""","'") & ""","
      & """RefreshTime"":""" & IF(ISBLANK(Nodes[RefreshTime]),"",
          FORMAT(Nodes[RefreshTime],"YYYY-MM-DDTHH:mm:SS")&"Z") & ""","
      & """RefreshStatus"":"""& IF(ISBLANK(Nodes[RefreshStatus]),"",
          Nodes[RefreshStatus]) & ""","
      & """PbiUrl"":"""   & IF(ISBLANK(Nodes[PbiUrl]),"",Nodes[PbiUrl]) & """"
    & "}", "," )
RETURN "[" & _rows & "]"
DAX — TOJSON (Power BI May 2024+) 
NodesJSON_TOJSON =
TOJSON(
  SELECTCOLUMNS( Nodes,
    "NodeId",Nodes[NodeId], "NodeName",Nodes[NodeName],
    "NodeType",Nodes[NodeType], "Workspace",Nodes[Workspace],
    "RefreshTime",IF(ISBLANK(Nodes[RefreshTime]),BLANK(),
      FORMAT(Nodes[RefreshTime],"yyyy-MM-ddTHH:mm:ss")&"Z"),
    "RefreshStatus",IF(ISBLANK(Nodes[RefreshStatus]),BLANK(),Nodes[RefreshStatus]),
    "PbiUrl",IF(ISBLANK(Nodes[PbiUrl]),BLANK(),Nodes[PbiUrl])
  ), 100000 -- maxCapacity
)
4

Create the EdgesJSON Measure

Same pattern — converts your Edges table to JSON:

DAX 
EdgesJSON =
VAR _rows =
  CONCATENATEX( Edges,
    "{"
      & """SourceId"":""" & SUBSTITUTE([SourceId],"""","'") & ""","
      & """TargetId"":""" & SUBSTITUTE([TargetId],"""","'") & """"
    & "}", "," )
RETURN "[" & _rows & "]"

-- OR with TOJSON (Power BI May 2024+):
EdgesJSON_TOJSON = TOJSON(SELECTCOLUMNS(Edges,"SourceId",Edges[SourceId],"TargetId",Edges[TargetId]),100000)
5

Map to Field Wells

Add the PBI Lineage Explorer visual to your report. Drag NodesJSON → Nodes JSON field well | EdgesJSON → Edges JSON field well. The graph renders instantly ✨

Data Schema

Two flat tables — Nodes & Edges

Keep them as separate tables. Do not union or append them.

🌐 Nodes Table
ColumnTypeRequired
NodeIdTextYes
NodeNameTextYes
NodeTypeTextYes
WorkspaceTextYes
RefreshTimeDateTimeNo
RefreshStatusTextNo
PbiUrlTextNo
🔗 Edges Table
ColumnTypeRequired
SourceIdTextYes
TargetIdTextYes
⚠️ SourceId / TargetId must exactly match NodeId values — case-sensitive.
Edges: Dataflow → Dataset | Dataset → Report | Dataflow → Dataflow (chained)

FAQ & Troubleshooting

Common questions

What Power BI versions are supported?
The visual works with all Power BI Desktop and Service versions. The CONCATENATEX DAX approach works with every version. The TOJSON approach requires Power BI May 2024 or later.
TOJSON is only returning a few rows — what's wrong?
You must pass the maxCapacity second argument: TOJSON(..., 100000). Without it, the default cap is very small and data gets silently truncated.
API returns 401 / Unauthorized
In Power Query go to Data Source Settings → Edit Credentials → Organizational account. Make sure the account has access to the workspaces you want to scan.
No edges are visible in the graph
Check that SourceId / TargetId values exactly match NodeId values in the Nodes table. The match is case-sensitive.
RefreshStatus badges are not showing
Values must be lowercase: success, failed, or progress. The visual automatically lower-cases on import, so Success, SUCCESS, etc. will all work.
Dataset → Dataflow edge is missing
Not all datasets expose their dataflow via the standard REST API. Use the Admin Scanner API approach for complete cross-workspace lineage including chained dataflows.
Graph is loading slowly with many nodes
Filter your workspaces in the Power Query M code using $filter=name eq 'My Workspace' to reduce the dataset size. You can also use the Compact view toggle.
Can I build from source?
Yes! Requires Node.js 18+ and npm. Run npm install, then npm run start for dev server (hot reload at localhost:8080) or npm run package to build the .pbiviz file.
What keyboard shortcuts are available?
Ctrl+F focuses the search box to filter nodes by name. Esc clears the current selection, search query, and Failed-Only filter in one press.
